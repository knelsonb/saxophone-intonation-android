## New invariants surfaced by council

| # | Invariant | Source |
|---|---|---|
| U21 | `useMidiBus` listeners MUST be invoked synchronously in the same JS call stack as `handle.noteOn/Off/programChange/etc.` — NOT via microtask, setTimeout, EventEmitter-with-defer, or any queue. The synchronous contract is part of the bus's published TypeScript interface and enforced in CI tests. | F18-15 council ruling + Legolas/Aragorn synchrony warnings. |
| U22 | `metroProfilesJson` deserialization on cold launch must validate before overwriting v1.2 legacy fields. Validation failure falls back silently to legacy fields. | F7 Aragorn hardening. |
| U23 | Channel-claim conflicts NEVER surface a user-visible error toast. The bus falls back to its WAV path silently and emits a forensic-log event (`bus.warn('channel-claim-denied', {role, attemptedBy})`) for debugging. | F18-16 Aragorn hardening. |
| U24 | Render-count CI assertions are a v1.3.0 ship-blocker. SetupScreen must fire zero re-renders on a synthetic audio-buffer event; MetroScreen must fire zero re-renders on a non-metro-state update. | G7 unanimous council ruling. |
| U25 | Pipes default voice across the codebase = **Synth Lead Square (GM 80)** per user directive. Both v1.3 design docs' references to "Choir Aahs / GM 52" are stale and must update during implementation. | User directive, supersedes Gandalf's design-doc default. |

## Migration order (locked)

Per Gandalf's recommended sequence, each step independently buildable + testable:

1. **Introduce `useMidiBus`** — bus alone, no consumers migrated yet. Owns synth singleton lifecycle, channel-role enum, WAV fallback path, master-mute, forensic-log integration. Synchronous-listener contract codified in the TypeScript interface + first-pass CI tests.
2. **Migrate `useDrone`** to the bus — drone reserves `'drone'` channel, no direct synth calls. Verify no regressions on the v1.0.1 drone-mute coordination.
3. **Migrate `useMetronome`** to the bus — drums channel claim, fall-back path moves to bus ownership. WAV fallback continues during synth warm-up via the bus.
4. **Migrate pipes** to the bus — new `usePitchPipes` hook, `'pipes'` channel, infinite-sustain, A4 compensation via baseline pitch-bend.
5. **Introduce `useUiPrefsStore`** + `useDronePrefs` — extract UI prefs out of useAudioEngine; debounced writer; per-field setters + update() escape hatch.
6. **Migrate screens** to consume the new hook shapes — also lands the v1.3 redesign (METRO + TUNER top-pill nav, 4 user profile slots, accordion CUSTOMIZATION).
7. **Engine split** — shatter `useAudioEngine` into `useAudioStream`, `usePitchPipeline`, `useCollectBuckets`. Coordinator-owned shared refs.
8. **Delete `useAudioEngine` wrapper + `pitchTones.ts`** after one-release `@deprecated` period (v1.4.0 work).
